Envie o ficheiro .sql gerado por “Baixar dump completo” ou um pg_dump em formato plain SQL. O conteúdo será aplicado com psql -f na base configurada no Django.

Dump completo de produção (todos os dados): se a base local já correu migrate, as tabelas em public já existem e o CREATE TABLE do dump falha; o psql para no primeiro erro (ON_ERROR_STOP) e os COPY/INSERT seguintes não executam. Para o dump recriar tabelas e inserir tudo, marque a opção abaixo (ou defina PG_RESTORE_RESET_PUBLIC_SCHEMA=true no .env{% if pg_restore_reset_public_env %}; neste ambiente essa variável já está ativa — o import repõe public mesmo sem marcar a caixa{% endif %}). Isto apaga todo o conteúdo do schema public antes do ficheiro; só faça se o dump for uma cópia completa da mesma base.

Antes do ficheiro, o restore executa também DROP SCHEMA IF EXISTS … CASCADE para schemas que muitos dumps voltam a recriar (por defeito: _heroku, heroku_ext, rede-gov, tst). Pode acrescentar mais nomes no .env com PG_RESTORE_PREAMBLE_EXTRA_SCHEMAS=outro_um,outro_dois (só letras, números, _ e -). Isto evita erros do tipo “schema … already exists” em re-importações.
